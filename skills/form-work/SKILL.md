# Паттерны модуля формы

## ВАЖНО: Когда применять этот навык

Применяй ВСЕГДА когда:
- Пишешь код в модуле управляемой формы
- Добавляешь обработчики событий элементов формы
- Делаешь серверные вызовы из клиентского кода
- Работаешь с табличными частями на форме
- Открываешь другие формы или показываешь диалоги

## Критический принцип

Каждый вызов `&НаСервере` = сериализация ВСЕЙ формы + сетевой round-trip.
Для формы с 1000 строк в табличной части это мегабайты данных туда-обратно.
МИНИМИЗИРУЙ количество и объём серверных вызовов.

## ШАБЛОНЫ КОДА (копируй и адаптируй)

### Шаблон 1: Батчинг серверных вызовов
```bsl
// КЛИЕНТ: один вызов для всех данных
&НаКлиенте
Процедура КонтрагентПриИзменении(Элемент)
    ДанныеЗаполнения = ПолучитьВсеДанныеОдинВызов(Объект.Контрагент, Объект.Дата);
    
    Объект.Договор = ДанныеЗаполнения.Договор;
    Объект.Валюта = ДанныеЗаполнения.Валюта;
    КурсВалюты = ДанныеЗаполнения.Курс;
КонецПроцедуры

// СЕРВЕР БЕЗ КОНТЕКСТА: все данные за один раз
&НаСервереБезКонтекста
Функция ПолучитьВсеДанныеОдинВызов(КонтрагентСсылка, ДатаДокумента)
    Результат = Новый Структура("Договор, Валюта, Курс");
    
    РеквизитыКонтрагента = ОбщегоНазначения.ЗначенияРеквизитовОбъекта(
        КонтрагентСсылка, "ДоговорПоУмолчанию");
    
    Результат.Договор = РеквизитыКонтрагента.ДоговорПоУмолчанию;
    
    Если ЗначениеЗаполнено(Результат.Договор) Тогда
        ВалютаДоговора = ОбщегоНазначения.ЗначениеРеквизитаОбъекта(
            Результат.Договор, "ВалютаВзаиморасчетов");
        Результат.Валюта = ВалютаДоговора;
        Результат.Курс = РаботаСКурсамиВалют.ПолучитьКурсВалюты(ВалютаДоговора, ДатаДокумента);
    КонецЕсли;
    
    Возврат Результат;
КонецФункции
```

### Шаблон 2: Асинхронный диалог с callback
```bsl
// Команда с подтверждением
&НаКлиенте
Процедура УдалитьСтроку(Команда)
    ПоказатьВопрос(
        Новый ОписаниеОповещения("ПослеПодтверждения", ЭтотОбъект),
        НСтр("ru = 'Удалить выбранную строку?'"),
        РежимДиалогаВопрос.ДаНет);
КонецПроцедуры

// Callback после ответа
&НаКлиенте
Процедура ПослеПодтверждения(Результат, ДополнительныеПараметры) Экспорт
    Если Результат = КодВозвратаДиалога.Да Тогда
        // Выполняем действие
        Объект.Товары.Удалить(Элементы.Товары.ТекущаяСтрока);
    КонецЕсли;
КонецПроцедуры
```

### Шаблон 3: Преобразование данных формы
```bsl
&НаСервере
Процедура ВыполнитьДействиеНаСервере()
    // Преобразуем данные формы в объект
    ДокументОбъект = РеквизитФормыВЗначение("Объект");
    
    // Вызываем методы модуля объекта
    ДокументОбъект.ЗаполнитьПоУмолчанию();
    
    // ОБЯЗАТЕЛЬНО преобразуем обратно!
    ЗначениеВРеквизитФормы(ДокументОбъект, "Объект");
КонецПроцедуры
```

### Шаблон 4: Открытие формы с параметрами и callback
```bsl
&НаКлиенте
Процедура ОткрытьФормуПодбора(Команда)
    ПараметрыФормы = Новый Структура;
    ПараметрыФормы.Вставить("Организация", Объект.Организация);
    ПараметрыФормы.Вставить("МножественныйВыбор", Истина);
    
    ОткрытьФорму("Обработка.Подбор.Форма",
        ПараметрыФормы,
        ЭтотОбъект,
        ,,,
        Новый ОписаниеОповещения("ПослеПодбора", ЭтотОбъект));
КонецПроцедуры

&НаКлиенте
Процедура ПослеПодбора(Результат, ДополнительныеПараметры) Экспорт
    Если Результат = Неопределено Тогда
        Возврат;
    КонецЕсли;
    
    Для Каждого Элемент Из Результат Цикл
        НоваяСтрока = Объект.Товары.Добавить();
        ЗаполнитьЗначенияСвойств(НоваяСтрока, Элемент);
    КонецЦикла;
КонецПроцедуры
```

## Правило 1: Иерархия директив — предпочитайте безконтекстные вызовы

**Почему:** Выбор директивы определяет, какие данные передаются между клиентом и сервером. Неправильный выбор приводит к избыточному трафику.

### Порядок предпочтения (от лучшего к худшему)

```
1. &НаКлиентеНаСервереБезКонтекста  — чистые вычисления, работает в обеих средах
2. &НаСервереБезКонтекста            — обращение к БД, без контекста формы
3. &НаСервере                        — нужен доступ к реквизитам формы
4. &НаКлиенте                        — интерактивная логика (диалоги, навигация)
```

### Правило принятия решения

```
Нужно ли обращаться к базе данных?
├── Нет → &НаКлиенте или &НаКлиентеНаСервереБезКонтекста
└── Да →
    Нужен ли доступ к реквизитам формы (чтение или изменение)?
    ├── Нет → &НаСервереБезКонтекста (передаём только параметры)
    └── Да → &НаСервере (передаётся весь контекст)
```

### Пример: выбор правильной директивы

```bsl
// Чистое вычисление — обе среды, никаких вызовов сервера
&НаКлиентеНаСервереБезКонтекста
Функция РассчитатьСумму(Количество, Цена, СтавкаНДС)
    СуммаБезНДС = Количество * Цена;
    СуммаНДС = СуммаБезНДС * СтавкаНДС / 100;
    Возврат СуммаБезНДС + СуммаНДС;
КонецФункции

// Нужны данные из БД, но НЕ нужны реквизиты формы
&НаСервереБезКонтекста
Функция ПолучитьДанныеНоменклатуры(НоменклатураСсылка)
    Возврат ОбщегоНазначения.ЗначенияРеквизитовОбъекта(
        НоменклатураСсылка,
        "Наименование, ЕдиницаИзмерения, СтавкаНДС, Цена");
КонецФункции

// Нужен доступ к реквизитам формы для их изменения
&НаСервере
Процедура ПересчитатьИтогиНаСервере()
    // Пересчитываем итоговые реквизиты формы на основе табличной части
    Объект.СуммаДокумента = Объект.Товары.Итог("Сумма");
    Объект.СуммаНДС = Объект.Товары.Итог("СуммаНДС");
КонецПроцедуры
```

## Правило 2: Минимизация серверных вызовов — батчинг данных

**Почему:** Каждый серверный вызов — это:
1. Сериализация параметров (или контекста формы для `&НаСервере`)
2. HTTP-запрос (50-200 мс на WAN)
3. Десериализация на сервере
4. Выполнение
5. Сериализация результата
6. HTTP-ответ
7. Десериализация на клиенте

Три последовательных вызова × 100 мс = **300 мс задержка**, заметная пользователю. Один вызов, возвращающий три результата, — 100 мс.

### Правильно — один вызов, все данные

```bsl
&НаКлиенте
Процедура КонтрагентПриИзменении(Элемент)
    // Один серверный вызов — все данные
    ДанныеЗаполнения = ПолучитьДанныеЗаполненияПоКонтрагенту(Объект.Контрагент, Объект.Дата);
    
    Объект.Договор = ДанныеЗаполнения.Договор;
    Объект.Организация = ДанныеЗаполнения.Организация;
    Объект.Валюта = ДанныеЗаполнения.Валюта;
    КурсВалюты = ДанныеЗаполнения.Курс;
КонецПроцедуры

&НаСервереБезКонтекста
Функция ПолучитьДанныеЗаполненияПоКонтрагенту(КонтрагентСсылка, ДатаДокумента)
    
    Результат = Новый Структура;
    
    // Все данные собираются за один вызов
    РеквизитыКонтрагента = ОбщегоНазначения.ЗначенияРеквизитовОбъекта(
        КонтрагентСсылка, "ДоговорПоУмолчанию, ОсновнаяОрганизация");
    
    Результат.Вставить("Договор", РеквизитыКонтрагента.ДоговорПоУмолчанию);
    Результат.Вставить("Организация", РеквизитыКонтрагента.ОсновнаяОрганизация);
    
    Если ЗначениеЗаполнено(РеквизитыКонтрагента.ДоговорПоУмолчанию) Тогда
        ВалютаДоговора = ОбщегоНазначения.ЗначениеРеквизитаОбъекта(
            РеквизитыКонтрагента.ДоговорПоУмолчанию, "ВалютаВзаиморасчетов");
        Результат.Вставить("Валюта", ВалютаДоговора);
        Результат.Вставить("Курс", РаботаСКурсамиВалют.ПолучитьКурсВалюты(ВалютаДоговора, ДатаДокумента));
    Иначе
        Результат.Вставить("Валюта", Неопределено);
        Результат.Вставить("Курс", 1);
    КонецЕсли;
    
    Возврат Результат;
    
КонецФункции
```

### Неправильно — три отдельных вызова

```bsl
// ❌ ПЛОХО: три round-trip
&НаКлиенте
Процедура КонтрагентПриИзменении(Элемент)
    Объект.Договор = ПолучитьДоговор(Объект.Контрагент);       // вызов 1
    Объект.Валюта = ПолучитьВалюту(Объект.Договор);             // вызов 2
    КурсВалюты = ПолучитьКурс(Объект.Валюта, Объект.Дата);      // вызов 3
КонецПроцедуры
```

## Правило 3: ДанныеФормыВЗначение / ЗначениеВДанныеФормы — модель данных формы

**Почему:** На сервере формы данные объекта (справочника, документа) хранятся не как настоящий объект, а как `ДанныеФормы*` — специальная обёртка платформы. Чтобы вызвать методы объекта (Записать, Провести, вызвать процедуры модуля объекта), нужно преобразование.

### Когда нужно преобразование

| Сценарий | Нужно? | Метод |
|----------|--------|-------|
| Чтение реквизитов формы | Нет — `Объект.Реквизит` работает напрямую |
| Вызов методов модуля объекта | Да | `ДанныеФормыВЗначение()` + `ЗначениеВДанныеФормы()` |
| Работа с табличными частями | Обычно нет — `Объект.ТабличнаяЧасть` доступна |
| Передача объекта в общий модуль | Да — общие модули работают с настоящими объектами |

### Правильно — преобразование для вызова методов

```bsl
&НаСервере
Процедура ЗаполнитьПоУмолчаниюНаСервере()
    
    // Преобразуем данные формы в объект
    ДокументОбъект = РеквизитФормыВЗначение("Объект");
    
    // Теперь можно вызывать методы модуля объекта
    ДокументОбъект.ЗаполнитьТоварыПоУмолчанию();
    
    // Преобразуем обратно — иначе изменения не отразятся на форме!
    ЗначениеВРеквизитФормы(ДокументОбъект, "Объект");
    
КонецПроцедуры
```

### Типичная ошибка — забыли ЗначениеВРеквизитФормы

```bsl
// ❌ ПЛОХО: изменения потеряны!
&НаСервере
Процедура ЗаполнитьНаСервере()
    ДокументОбъект = РеквизитФормыВЗначение("Объект");
    ДокументОбъект.ЗаполнитьТаблицу();
    // ЗАБЫЛИ: ЗначениеВРеквизитФормы(ДокументОбъект, "Объект");
    // Все изменения пропали! Форма показывает старые данные.
КонецПроцедуры
```

## Правило 4: Обработчики событий формы — порядок и назначение

**Почему:** Платформа 1С вызывает обработчики событий в определённом порядке. Понимание порядка критически важно — код в неправильном обработчике либо не выполнится, либо выполнится с устаревшими данными.

### Порядок событий при открытии формы

```
1. ПриСозданииНаСервере     — форма создаётся на сервере, данные загружены
2. ПриОткрытии              — форма отобразилась на клиенте (клиентский код)
```

### Порядок событий при записи

```
1. ПередЗаписьюНаКлиенте    — клиент: можно отменить запись (Отказ = Истина)
2. ПередЗаписьюНаСервере     — сервер: последняя проверка перед записью
3. ПриЗаписиНаСервере        — сервер: дополнительная логика после записи объекта
4. ПослеЗаписиНаСервере      — сервер: обновление формы после записи
5. ПослеЗаписи               — клиент: оповещение об успешной записи
```

### Пример: разнесение логики по обработчикам

```bsl
// ПриСозданииНаСервере — инициализация формы
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    
    Если Параметры.Свойство("ЗначенияЗаполнения") Тогда
        ЗаполнитьЗначенияСвойств(Объект, Параметры.ЗначенияЗаполнения);
    КонецЕсли;
    
    // Условная видимость элементов
    Элементы.ГруппаСклад.Видимость = (Объект.ВидОперации <> 
        Перечисления.ВидыОпераций.Услуга);
    
    // Заполнение списков выбора
    ОбновитьСписокВыбораДоговоров();
    
КонецПроцедуры

// ПередЗаписьюНаКлиенте — проверки, требующие интерактивности
&НаКлиенте
Процедура ПередЗаписью(Отказ, ПараметрыЗаписи)
    
    // Проверка, требующая подтверждения пользователя
    Если Объект.СуммаДокумента > 1000000 Тогда
        Если Не ПараметрыЗаписи.Свойство("ПодтверждениеСуммы") Тогда
            Отказ = Истина;
            ПоказатьВопрос(
                Новый ОписаниеОповещения("ПослеПодтвержденияСуммы", ЭтотОбъект, ПараметрыЗаписи),
                НСтр("ru = 'Сумма документа превышает 1 000 000. Продолжить?'"),
                РежимДиалогаВопрос.ДаНет);
        КонецЕсли;
    КонецЕсли;
    
КонецПроцедуры

// ПриЗаписиНаСервере — действия после записи объекта, но в той же транзакции
&НаСервере
Процедура ПриЗаписиНаСервере(Отказ, ТекущийОбъект, ПараметрыЗаписи)
    
    // ТекущийОбъект — это настоящий объект (не данные формы!)
    // Можно вызывать методы модуля объекта, писать в регистры
    
    Если ТекущийОбъект.ЭтоНовый() Тогда
        // Дополнительная инициализация при первой записи
    КонецЕсли;
    
КонецПроцедуры

// ПослеЗаписи — уведомления на клиенте
&НаКлиенте
Процедура ПослеЗаписи(ПараметрыЗаписи)
    // Оповещаем другие открытые формы об изменении
    Оповестить("Запись_РеализацияТоваровУслуг", 
        Новый Структура("Ссылка", Объект.Ссылка), ЭтотОбъект);
КонецПроцедуры
```

## Правило 5: Динамические списки — настройка и оптимизация

**Почему:** Динамический список — основной способ отображения больших данных (справочники, документы, регистры). Он автоматически реализует пагинацию, поиск, сортировку. Неправильная настройка приводит к медленной загрузке и зависанию формы.

### Правильная настройка в ПриСозданииНаСервере

```bsl
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    
    // Программная настройка динамического списка
    Список = Элементы.СписокДокументов;
    
    // Установка отбора через параметры (НЕ через фиксированные настройки!)
    ОбщегоНазначенияКлиентСервер.УстановитьЭлементОтбораДинамическогоСписка(
        СписокДокументов,
        "Организация",
        Организация,
        ВидСравненияКомпоновкиДанных.Равно,
        ,
        ЗначениеЗаполнено(Организация));
    
КонецПроцедуры
```

### Произвольный запрос для динамического списка

```bsl
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    
    // Произвольный запрос — когда нужно показать данные из нескольких таблиц
    СписокДокументов.ПроизвольныйЗапрос = Истина;
    СписокДокументов.ТекстЗапроса =
    "ВЫБРАТЬ
    |   Реализация.Ссылка,
    |   Реализация.Дата,
    |   Реализация.Номер,
    |   Реализация.Контрагент,
    |   Реализация.СуммаДокумента,
    |   ЕСТЬNULL(Задолженность.СуммаОстаток, 0) КАК ОстатокЗадолженности
    |ИЗ
    |   Документ.РеализацияТоваровУслуг КАК Реализация
    |       ЛЕВОЕ СОЕДИНЕНИЕ РегистрНакопления.ВзаиморасчетыСКонтрагентами.Остатки КАК Задолженность
    |       ПО Реализация.Контрагент = Задолженность.Контрагент
    |           И Реализация.Договор = Задолженность.Договор
    |{ГДЕ
    |   Реализация.Дата >= &ДатаНачала}";
    
    СписокДокументов.Параметры.УстановитьЗначениеПараметра("ДатаНачала", НачалоМесяца(ТекущаяДатаСеанса()));
    
КонецПроцедуры
```

### Правила работы с динамическими списками

1. **Не загружайте все данные** — динамический список сам реализует пагинацию
2. **Отборы через КомпоновкаДанных** — не через WHERE в запросе (пользователь сможет изменять отборы)
3. **Индексируйте поля сортировки** — динамический список часто сортирует по дате, номеру
4. **Не используйте УПОРЯДОЧИТЬ ПО** в произвольном запросе — сортировка задаётся настройками списка

## Правило 6: Условное оформление — программная настройка

**Почему:** Условное оформление позволяет визуально выделять строки формы (цветом, шрифтом) на основе данных. Программная настройка в `ПриСозданииНаСервере` предпочтительнее настройки в конфигураторе, потому что:
- Можно задать динамические условия (зависящие от параметров)
- Проще поддерживать — код vs визуальный конструктор
- Работает с произвольными запросами

### Пример: подсветка просроченных документов

```bsl
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    
    УстановитьУсловноеОформление();
    
КонецПроцедуры

&НаСервере
Процедура УстановитьУсловноеОформление()
    
    УсловноеОформление.Элементы.Очистить();
    
    // Правило 1: просроченные документы — красным фоном
    ЭлементУО = УсловноеОформление.Элементы.Добавить();
    
    // Условие: дата оплаты < текущая дата И статус = "Не оплачен"
    ЭлементОтбора = ЭлементУО.Отбор.Элементы.Добавить(Тип("ЭлементОтбораКомпоновкиДанных"));
    ЭлементОтбора.ЛевоеЗначение = Новый ПолеКомпоновкиДанных("ДатаОплаты");
    ЭлементОтбора.ВидСравнения = ВидСравненияКомпоновкиДанных.Меньше;
    ЭлементОтбора.ПравоеЗначение = ТекущаяДатаСеанса();
    ЭлементОтбора.Использование = Истина;
    
    ЭлементОтбора2 = ЭлементУО.Отбор.Элементы.Добавить(Тип("ЭлементОтбораКомпоновкиДанных"));
    ЭлементОтбора2.ЛевоеЗначение = Новый ПолеКомпоновкиДанных("Оплачен");
    ЭлементОтбора2.ВидСравнения = ВидСравненияКомпоновкиДанных.Равно;
    ЭлементОтбора2.ПравоеЗначение = Ложь;
    ЭлементОтбора2.Использование = Истина;
    
    // Оформление: красный фон
    ЭлементУО.Оформление.УстановитьЗначениеПараметра("ЦветФона", WebЦвета.МисттиРоуз);
    
    // Область применения — вся строка таблицы
    ПолеОформления = ЭлементУО.Поля.Элементы.Добавить();
    ПолеОформления.Поле = Новый ПолеКомпоновкиДанных("СписокДокументов");
    
КонецПроцедуры
```

## Правило 7: Обработчики команд и асинхронные вызовы

**Почему:** В управляемом приложении модальные вызовы (`Предупреждение()`, `Вопрос()`, `ОткрытьФормуМодально()`) **запрещены** в веб-клиенте и нежелательны в тонком клиенте. Вместо них нужно использовать асинхронные аналоги с `ОписаниеОповещения`.

**Стандарт ИТС:** «Ограничения на использование модальных методов».

### Правильно — асинхронный подход

```bsl
// Команда удаления — с подтверждением через асинхронный диалог
&НаКлиенте
Процедура УдалитьСтрокуТоваров(Команда)
    
    Если Элементы.Товары.ТекущаяСтрока = Неопределено Тогда
        Возврат;
    КонецЕсли;
    
    // Асинхронный вопрос с callback
    ПоказатьВопрос(
        Новый ОписаниеОповещения("ПослеПодтвержденияУдаления", ЭтотОбъект),
        НСтр("ru = 'Удалить выбранную строку?'"),
        РежимДиалогаВопрос.ДаНет,
        ,
        КодВозвратаДиалога.Нет);  // По умолчанию — «Нет»
    
КонецПроцедуры

// Callback — вызывается после ответа пользователя
&НаКлиенте
Процедура ПослеПодтвержденияУдаления(Результат, ДополнительныеПараметры) Экспорт
    
    Если Результат = КодВозвратаДиалога.Да Тогда
        Объект.Товары.Удалить(Элементы.Товары.ТекущаяСтрока);
    КонецЕсли;
    
КонецПроцедуры
```

### Неправильно — модальный вызов

```bsl
// ❌ ПЛОХО: модальный вызов — не работает в веб-клиенте
&НаКлиенте
Процедура УдалитьСтрокуТоваров(Команда)
    
    Ответ = Вопрос("Удалить строку?", РежимДиалогаВопрос.ДаНет); // МОДАЛЬНЫЙ — ЗАПРЕЩЕНО!
    Если Ответ = КодВозвратаДиалога.Да Тогда
        Объект.Товары.Удалить(Элементы.Товары.ТекущаяСтрока);
    КонецЕсли;
    
КонецПроцедуры
```

## Правило 8: Открытие форм и обработка результата

**Почему:** Правильная передача параметров при открытии формы и обработка результата закрытия — основа навигации в 1С. Неправильная работа с параметрами приводит к потере данных и некорректному поведению.

### Открытие формы с параметрами

```bsl
&НаКлиенте
Процедура ОткрытьФормуПодбораТоваров(Команда)
    
    ПараметрыФормы = Новый Структура;
    ПараметрыФормы.Вставить("Организация", Объект.Организация);
    ПараметрыФормы.Вставить("Склад", Объект.Склад);
    ПараметрыФормы.Вставить("Дата", Объект.Дата);
    ПараметрыФормы.Вставить("МножественныйВыбор", Истина);
    
    ОткрытьФорму("Обработка.ПодборТоваров.Форма",
        ПараметрыФормы,
        ЭтотОбъект,                     // Владелец
        ,                                 // Уникальность
        ,                                 // Окно
        ,                                 // НавигационнаяСсылка
        Новый ОписаниеОповещения("ПослеПодбораТоваров", ЭтотОбъект));
    
КонецПроцедуры

// Обработка результата закрытия формы подбора
&НаКлиенте
Процедура ПослеПодбораТоваров(РезультатПодбора, ДополнительныеПараметры) Экспорт
    
    Если РезультатПодбора = Неопределено Тогда
        Возврат; // Пользователь закрыл форму без выбора
    КонецЕсли;
    
    // РезультатПодбора — массив структур с данными товаров
    Для Каждого ДанныеТовара Из РезультатПодбора Цикл
        НоваяСтрока = Объект.Товары.Добавить();
        ЗаполнитьЗначенияСвойств(НоваяСтрока, ДанныеТовара);
    КонецЦикла;
    
    Модифицированность = Истина; // Помечаем форму как изменённую
    
КонецПроцедуры
```

## Правило 9: Управление видимостью и доступностью элементов

**Почему:** Скрытие/показ элементов формы программно — стандартный способ адаптации интерфейса под бизнес-логику. Но неправильное управление (слишком много серверных вызовов, изменение видимости из клиентского обработчика без батчинга) приводит к «дёрганию» интерфейса и лишним вызовам сервера.

### Правильно — настройка при создании и по событиям

```bsl
// При создании — начальная настройка
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    
    УправлениеВидимостью();
    
КонецПроцедуры

// Общая процедура управления видимостью — вызывается из разных мест
&НаСервере
Процедура УправлениеВидимостью()
    
    ЭтоУслуга = Объект.ВидОперации = Перечисления.ВидыОпераций.Услуга;
    
    // Группируем изменения — все в одном серверном вызове
    Элементы.Склад.Видимость = НЕ ЭтоУслуга;
    Элементы.ТоварыКоличество.Видимость = НЕ ЭтоУслуга;
    Элементы.ТоварыЕдиницаИзмерения.Видимость = НЕ ЭтоУслуга;
    Элементы.ГруппаДоставка.Видимость = НЕ ЭтоУслуга;
    
    // Доступность
    Элементы.Контрагент.ТолькоПросмотр = Объект.Проведен;
    
КонецПроцедуры

// При изменении вида операции
&НаКлиенте
Процедура ВидОперацииПриИзменении(Элемент)
    // Один серверный вызов для всех изменений видимости
    УправлениеВидимостью();
КонецПроцедуры
```

### Неправильно — изменение видимости по одному элементу

```bsl
// ❌ ПЛОХО: 4 серверных вызова вместо одного
&НаКлиенте
Процедура ВидОперацииПриИзменении(Элемент)
    УстановитьВидимостьСклада();     // вызов 1
    УстановитьВидимостьКоличества(); // вызов 2
    УстановитьВидимостьЕдиницы();    // вызов 3
    УстановитьВидимостьДоставки();   // вызов 4
КонецПроцедуры
```

## Правило 10: Работа с табличными частями на форме

**Почему:** Табличные части — основной способ хранения множественных данных (строки документов, контакты контрагентов). Типичные ошибки: пересчёт в цикле с серверными вызовами, некорректная работа с текущей строкой, потеря данных при редактировании.

### Пересчёт строки — на клиенте, если возможно

```bsl
// Изменение количества — пересчёт суммы можно сделать на клиенте
&НаКлиенте
Процедура ТоварыКоличествоПриИзменении(Элемент)
    ТекущаяСтрока = Элементы.Товары.ТекущиеДанные;
    ТекущаяСтрока.Сумма = РассчитатьСумму(
        ТекущаяСтрока.Количество, ТекущаяСтрока.Цена, ТекущаяСтрока.СтавкаНДС);
КонецПроцедуры

// Изменение номенклатуры — нужен сервер для получения данных
&НаКлиенте
Процедура ТоварыНоменклатураПриИзменении(Элемент)
    ТекущаяСтрока = Элементы.Товары.ТекущиеДанные;
    ДанныеНоменклатуры = ПолучитьДанныеНоменклатуры(ТекущаяСтрока.Номенклатура);
    
    // Заполняем всё на клиенте из полученных данных
    ТекущаяСтрока.ЕдиницаИзмерения = ДанныеНоменклатуры.ЕдиницаИзмерения;
    ТекущаяСтрока.Цена = ДанныеНоменклатуры.Цена;
    ТекущаяСтрока.СтавкаНДС = ДанныеНоменклатуры.СтавкаНДС;
    ТекущаяСтрока.Сумма = РассчитатьСумму(
        ТекущаяСтрока.Количество, ТекущаяСтрока.Цена, ТекущаяСтрока.СтавкаНДС);
КонецПроцедуры
```

### Обход табличной части — на клиенте

```bsl
// Пересчёт итога по табличной части — можно на клиенте
&НаКлиенте
Процедура ПересчитатьИтого()
    ИтогоСумма = 0;
    Для Каждого СтрокаТовара Из Объект.Товары Цикл
        ИтогоСумма = ИтогоСумма + СтрокаТовара.Сумма;
    КонецЦикла;
    
    // Обновляем реквизит формы (не объекта!) — для отображения итога
    ИтогоПоДокументу = ИтогоСумма;
КонецПроцедуры
```

## Правило 11: Оповещения между формами

**Почему:** Когда пользователь изменяет данные в одной форме, другие открытые формы должны обновиться. Механизм `Оповестить()` / `ОбработкаОповещения()` — стандартный способ межформенного взаимодействия.

### Паттерн: отправка оповещения после записи

```bsl
// Форма документа — после записи
&НаКлиенте
Процедура ПослеЗаписи(ПараметрыЗаписи)
    Оповестить("Запись_РеализацияТоваровУслуг",
        Новый Структура("Ссылка, Проведен", Объект.Ссылка, Объект.Проведен),
        ЭтотОбъект);
КонецПроцедуры

// Форма списка — обработка оповещения
&НаКлиенте
Процедура ОбработкаОповещения(ИмяСобытия, Параметр, Источник)
    
    Если ИмяСобытия = "Запись_РеализацияТоваровУслуг" Тогда
        // Обновляем динамический список
        Элементы.Список.Обновить();
    КонецЕсли;
    
КонецПроцедуры
```

## Правило 12: Передаваемые типы данных между клиентом и сервером

**Почему:** Не все типы данных можно передать между клиентом и сервером. Понимание ограничений предотвращает ошибки «Значение не может быть сериализовано».

### Типы, передаваемые свободно

| Тип | Пример |
|-----|--------|
| Примитивные | Строка, Число, Дата, Булево, Неопределено, NULL |
| Ссылки | СправочникСсылка.*, ДокументСсылка.* |
| Перечисления | ПеречислениеСсылка.* |
| Структура / Соответствие | Новый Структура, Новый Соответствие |
| Массив | Новый Массив |
| ФиксированныеКоллекции | ФиксированнаяСтруктура, ФиксированныйМассив |
| Хранилище значения | ХранилищеЗначения |

### Типы, НЕ передаваемые с клиента на сервер (и наоборот)

| Тип | Почему нельзя | Альтернатива |
|-----|--------------|-------------|
| ТаблицаЗначений | Серверный тип | ДанныеФормыКоллекция (через реквизиты формы) |
| ДеревоЗначений | Серверный тип | ДанныеФормыДерево (через реквизиты формы) |
| ОбъектМетаданных | Серверный тип | Передавать ИмяМетаданных (строку) |
| COMОбъект | Серверный тип | Не применимо |
| Запрос, РезультатЗапроса | Серверный тип | Передавать результат (структура/массив) |

### Пример: передача данных между средами

```bsl
// ✅ ПРАВИЛЬНО: возвращаем структуру (сериализуется)
&НаСервереБезКонтекста
Функция ПолучитьДанныеНаСервере(СсылкаНаОбъект)
    Возврат Новый Структура("Наименование, Код",
        СсылкаНаОбъект.Наименование, СсылкаНаОбъект.Код);
КонецФункции

// ❌ ПЛОХО: попытка вернуть ТаблицуЗначений — ошибка сериализации
&НаСервереБезКонтекста
Функция ПолучитьТаблицуНаСервере()
    Возврат Новый ТаблицаЗначений; // Ошибка! ТаблицаЗначений не передаётся на клиент
КонецФункции
```

## КОНТРОЛЬНЫЙ СПИСОК: Проверь перед коммитом

Используй этот список для проверки кода модулей форм:

### Серверные вызовы
- [ ] Используется `&НаСервереБезКонтекста` вместо `&НаСервере` где возможно
- [ ] Нет нескольких последовательных серверных вызовов (батчинг применён)
- [ ] Серверная функция возвращает ВСЕ нужные данные за один раз
- [ ] Нет серверных вызовов в циклах обработки данных

### Преобразование данных
- [ ] После `РеквизитФормыВЗначение()` есть `ЗначениеВРеквизитФормы()`
- [ ] Преобразование используется только когда нужны методы объекта
- [ ] Передаваемые типы данных сериализуемы (нет ТаблицаЗначений, Запрос и т.д.)

### Диалоги и навигация
- [ ] Используются асинхронные диалоги (`ПоказатьВопрос` с `ОписаниеОповещения`)
- [ ] Нет модальных вызовов (`Вопрос()`, `Предупреждение()`, `ОткрытьФормуМодально()`)
- [ ] Формы открываются через `ОткрытьФорму()` с callback
- [ ] Обработчики callback имеют суффикс `Экспорт`

### Обработчики событий
- [ ] Инициализация формы в `ПриСозданииНаСервере`
- [ ] Проверки с диалогами в `ПередЗаписьюНаКлиенте`
- [ ] Дополнительная логика записи в `ПриЗаписиНаСервере`
- [ ] Оповещения других форм в `ПослеЗаписи`
