# /subagent-dev — Делегированная/параллельная реализация задач (совместимость)

Этот скилл используется как совместимый вход для сценариев из внешних набора скиллов.
В текущем стеке параллельный запуск внешними субагентами не обеспечен автоматически, поэтому поведение строится как:

1) строгая декомпозиция задач;
2) последовательное выполнение по `tasks.md`;
3) валидация после каждого значимого шага.

## Вход

`$ARGUMENTS` — путь к `tasks.md` или `openspec/changes/<feature-id>`.

## Процесс

1. Прочитай `tasks.md`, затем `design.md`/`proposal.md` (если есть).
2. Разбей задачу на этапы:
   - независимые задачи (внутри одного блока),
   - зависимые задачи.
3. Выполняй задачи в порядке зависимостей через профильные скиллы.
4. После каждого этапа — валидация:
   - `/meta-validate` / `/cf-validate` / `/form-validate` / `/role-validate` / `/mxl-validate` в зависимости от типа;
   - для бизнес-логики и запросов применяй `validate_query`.
5. Обновляй статус в `tasks.md` (`[ ]` → `[x]`).
6. Финальный отчёт: выполнено/не выполнено, открытые риски, blocked tasks.

## Как использовать вместо полного автономного запуска

- Для небольших изменений: выполняй вручную или через `/openspec-apply`.
- Для крупных задач: `/openspec-apply` + `subagent-dev` для контроля ритма и фиксации отчётов.

## Критерий завершения

- нет незакрытых критичных задач;
- отчёт по каждой задаче содержит проверку и статус;
- артефакты (`tasks.md`, `design.md`, при наличии `proposal.md`) консистентны.
