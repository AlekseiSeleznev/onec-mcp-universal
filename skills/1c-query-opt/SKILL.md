# /1c-query-opt

Оркестратор оптимизации запросов. Не заменяет низкоуровневые инструменты, а задаёт корректную последовательность проверки.

## Вход

- Текст исходного запроса 1С
- Контекст (объект/отчёт/форма, ожидаемый объём данных)
- Симптомы (долго выполняется, блокировки, таймауты)

## Последовательность

1. Зафиксируй baseline:
   - входной запрос,
   - критерий успеха (например: уменьшить время >= 30%),
   - ограничение корректности результата (поля/отборы не теряем).
2. Сделай первичную диагностику:
   - примени паттерны из `/query-writing`,
   - найди анти-паттерны (лишние поля, ранние соединения, отсутствие ограничений).
3. Подготовь кандидат оптимизации (один атомарный вариант за итерацию).
4. Проверь синтаксис через `validate_query`.
5. Проверь корректность результата на тестовых параметрах через `execute_query`.
6. Сравни измерения до/после через `query_stats` (или эквивалентный фактический замер).
7. Зафиксируй итог: baseline, что изменили, эффект, риски.

## Правила

1. Нельзя одновременно менять логику и оптимизацию в одной итерации.
2. Любая оптимизация должна проходить `validate_query` и проверку результата.
3. Если эффект < целевого порога, зафиксируй как "no win" и предложи следующую гипотезу.
4. Всегда явно указывай компромиссы (читаемость, поддерживаемость, чувствительность к объёму данных).

## Выход

Краткий отчёт в формате:

- Baseline
- Candidate change
- Validation
- Measured effect
- Decision (accept/reject/next iteration)
