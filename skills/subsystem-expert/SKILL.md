# /subsystem-expert — Подсистемы 1С (совместимый вход)

Единая точка для операций с подсистемами:

- создание новой подсистемы (`/subsystem-compile`);
- изменение состава/свойств (`/subsystem-edit`);
- правка интерфейса команд (`/interface-edit`);
- проверка (`/subsystem-validate`).

## Процесс

1. Уточни тип операции:
   - `compile` — собрать объект через `/subsystem-compile`.
   - `edit` — изменить состав/структуру через `/subsystem-edit`.
   - `interface-edit` — правки интерфейса через `/interface-edit`.
2. Выполни нужный профильный скилл.
3. Заверши `/subsystem-validate`.
4. Если изменяются связанные роли/формы/объекты, дополняй проверку через `/validate`.

## Рекомендуемый порядок для крупных изменений

- `inspect` (по целевой подсистеме и конфигурации);
- `/subsystem-edit` или `/subsystem-compile`;
- `/interface-edit` (при работе с командным интерфейсом);
- `/subsystem-validate`.

## Правила

1. Перед запуском `/interface-edit` уточни целевую подсистему/`CommandInterface.xml`.
2. Не перехардкодь зависимости на `Subsystems/*` без проверки `inspect`.
3. После изменений всегда фиксируй влияние на связанные объекты и права.
