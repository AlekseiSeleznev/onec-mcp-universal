# /epf-build — Сборка обработки

## Usage

```
/epf-build <ProcessorName> [SrcDir] [OutDir]
```

| Параметр      | Обязательный | По умолчанию | Описание                             |
|---------------|:------------:|--------------|--------------------------------------|
| ProcessorName | да           | —            | Имя обработки (имя корневого XML)    |
| SrcDir        | нет          | `src`        | Каталог исходников                   |
| OutDir        | нет          | `build`      | Каталог для результата               |

## Параметры подключения (опционально)

Предпочтительно использовать конкретную базу — это надёжнее и не требует создания временной базы.

1. Прочитай `.v8-project.json` из корня проекта. Возьми `v8path` и разреши базу:
2. Если пользователь указал параметры подключения (путь, сервер) — используй напрямую
3. Если указал базу по имени — ищи по id / alias / name в `.v8-project.json`
4. Если не указал — сопоставь текущую ветку Git с `databases[].branches`
5. Если ветка не совпала — используй `default`
6. Если `.v8-project.json` нет или база не найдена — не указывай параметры подключения: скрипт автоматически создаст временную базу. Для EPF со ссылочными типами (CatalogRef, DocumentRef и т.д.) генерируются заглушки метаданных. Временная база удаляется после сборки.

**Автоопределение v8path (если не задан в `.v8-project.json` и не передан пользователем):**
- Windows: `C:\Program Files\1cv8\*\bin\1cv8.exe` — берётся последняя по сортировке версия
- Linux: `/opt/1cv8/x86_64/*/1cv8` — берётся последняя по сортировке версия

**Несовпадение версии клиента и сервера:** скрипт автоматически определяет версию сервера из лога ошибки и повторяет сборку с подходящей установленной версией. Если нужной версии нет — выводит список установленных.

**На Linux:** если доступен `xvfb-run` — используется автоматически (1cv8 требует дисплей).

Если использованная база не зарегистрирована — после выполнения предложи добавить через `/db-list add`.

## Команда

```powershell
python skills/epf-build/scripts/epf-build.py <параметры>
```

### Параметры скрипта

| Параметр | Обязательный | Описание |
|----------|:------------:|----------|
| `-V8Path <путь>` | нет | Каталог платформы или полный путь к бинарю (1cv8.exe на Windows, 1cv8 на Linux). Если не задан — автоопределение по установленным версиям. |
| `-InfoBasePath <путь>` | * | Файловая база |
| `-InfoBaseServer <сервер>` | * | Сервер 1С (для серверной базы) |
| `-InfoBaseRef <имя>` | * | Имя базы на сервере |
| `-UserName <имя>` | нет | Имя пользователя |
| `-Password <пароль>` | нет | Пароль |
| `-SourceFile <путь>` | да | Путь к корневому XML-файлу исходников |
| `-OutputFile <путь>` | да | Путь к выходному EPF/ERF-файлу |

> `*` — опционально. Если не указано — автоматически создаётся временная база со заглушками метаданных

## Примеры

```bash
# Linux — серверная база (версия клиента подбирается автоматически)
python3 skills/epf-build/scripts/epf-build.py \
  -InfoBaseServer "srv01" -InfoBaseRef "MyDB" \
  -SourceFile "src/МояОбработка.xml" -OutputFile "build/МояОбработка.epf"

# Linux — без базы (временная с заглушками)
python3 skills/epf-build/scripts/epf-build.py \
  -SourceFile "src/МояОбработка.xml" -OutputFile "build/МояОбработка.epf"
```

```powershell
# Windows — файловая база
python skills/epf-build/scripts/epf-build.py -InfoBasePath "C:\Bases\MyDB" -SourceFile "src/МояОбработка.xml" -OutputFile "build/МояОбработка.epf"

# Windows — серверная база
python skills/epf-build/scripts/epf-build.py -InfoBaseServer "srv01" -InfoBaseRef "MyDB" -UserName "Admin" -Password "secret" -SourceFile "src/МояОбработка.xml" -OutputFile "build/МояОбработка.epf"
```
