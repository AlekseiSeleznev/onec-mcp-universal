# /session-save

Сохраняет текущее состояние разработки в `session-notes.md` в корне проекта.

## Когда использовать

- При длинной сессии (риск переполнения контекста)
- Перед завершением работы
- Перед `/clear`

## Шаблон файла

```markdown
# Session Notes — <date> <time>

## Current Task
<текущая задача>

## Completed
- <что завершено>

## Pending
- <что осталось>

## Next Action
<конкретный следующий исполняемый шаг>

## Key Decisions
- <важные решения и почему>

## Modified Files
- <список изменённых файлов>
```

## Правила

1. `Next Action` должен быть конкретным и коротким.
2. В `Modified Files` включай все затронутые файлы.
3. Используй фактическое состояние, без планов «на будущее» в разделе Completed.
