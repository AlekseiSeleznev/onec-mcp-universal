# Паттерны работы с БСП

## ВАЖНО: Когда применять этот навык

Применяй ВСЕГДА когда:
- Работаешь с конфигурацией на базе БСП (УТ, ERP, БП, ЗУП)
- Нужна типовая функциональность (работа со строками, файлами, правами)
- Перед написанием новой утилитарной функции
- Работаешь с общими модулями

## Критический принцип

БСП = проверенный код на миллионах установок.
НЕ дублируй функционал БСП - используй готовые функции.
Перед написанием своей функции - ПРОВЕРЬ, есть ли она в БСП.

### Шаблон 1: Получение реквизитов объекта
```bsl
// ✅ ПРАВИЛЬНО: через БСП - один запрос
РеквизитыКонтрагента = ОбщегоНазначения.ЗначенияРеквизитовОбъекта(
    КонтрагентСсылка,
    "Наименование, ИНН, ОсновнойМенеджер");

Наименование = РеквизитыКонтрагента.Наименование;
ИНН = РеквизитыКонтрагента.ИНН;
Ответственный = РеквизитыКонтрагента.ОсновнойМенеджер;

// ❌ ПЛОХО: точечная нотация - три запроса
Наименование = КонтрагентСсылка.Наименование;
ИНН = КонтрагентСсылка.ИНН;
Ответственный = КонтрагентСсылка.ОсновнойМенеджер;
```

## Правило 1: Модуль ОбщегоНазначения — основной «швейцарский нож»

**Почему:** `ОбщегоНазначения` — центральный модуль БСП, содержащий сотни универсальных функций. Прежде чем писать свою реализацию, проверьте — возможно, в БСП уже есть готовая функция. Дублирование кода БСП — антипаттерн.

### Ключевые функции ОбщегоНазначения

| Функция | Описание | Когда использовать |
|---------|----------|-------------------|
| `ЗначениеРеквизитаОбъекта()` | Получает значение реквизита по ссылке | Вместо `Ссылка.Реквизит` (избегаем точечную нотацию) |
| `ЗначенияРеквизитовОбъекта()` | Получает несколько реквизитов по ссылке | Вместо нескольких обращений через точку |
| `СообщитьПользователю()` | Сообщение с привязкой к полю | Вместо `Сообщить()` |
| `МенеджерОбъектаПоСсылке()` | Менеджер по ссылке на объект | Вместо `Выполнить("Справочники." + Имя)` |
| `ПодсистемаСуществует()` | Проверка наличия подсистемы | Для условного вызова модулей |
| `ОбщийМодуль()` | Получение модуля по имени | Динамический вызов модуля БСП |
| `ЭтоСсылка()` | Проверка типа — это ссылка? | Валидация параметров |
| `СсылкаСуществует()` | Существует ли объект в БД? | Проверка перед обращением |

### Пример: ЗначенияРеквизитовОбъекта — вместо точечной нотации

```bsl
// ❌ ПЛОХО: три обращения к БД через точку
Наименование = КонтрагентСсылка.Наименование;      // чтение 1
ИНН = КонтрагентСсылка.ИНН;                          // чтение 2
Ответственный = КонтрагентСсылка.ОсновнойМенеджер;  // чтение 3

// ✅ ПРАВИЛЬНО: одно обращение через БСП
РеквизитыКонтрагента = ОбщегоНазначения.ЗначенияРеквизитовОбъекта(
    КонтрагентСсылка,
    "Наименование, ИНН, ОсновнойМенеджер");

Наименование = РеквизитыКонтрагента.Наименование;
ИНН = РеквизитыКонтрагента.ИНН;
Ответственный = РеквизитыКонтрагента.ОсновнойМенеджер;
```

### Пример: ЗначениеРеквизитаОбъекта — одно значение

```bsl
// ❌ ПЛОХО: обращение через точку
ВидНоменклатуры = НоменклатураСсылка.ВидНоменклатуры;

// ✅ ПРАВИЛЬНО: через БСП
ВидНоменклатуры = ОбщегоНазначения.ЗначениеРеквизитаОбъекта(
    НоменклатураСсылка, "ВидНоменклатуры");
```

## Правило 2: СтроковыеФункцииКлиентСервер — работа со строками

**Почему:** Модуль `СтроковыеФункцииКлиентСервер` (или `СтроковыеФункции` на сервере) содержит оптимизированные функции для работы со строками. Они корректно обрабатывают edge cases (пустые строки, разделители, Юникод) и проверены тысячами тестов.

### Ключевые функции

| Функция | Описание | Когда использовать |
|---------|----------|-------------------|
| `ПодставитьПараметрыВСтроку()` | Подстановка `%1`, `%2` в шаблон | Аналог `СтрШаблон()`, но с дополнительными проверками |
| `СтрокаСЧисломПредметов()` | «5 документов», «1 документ», «21 документ» | Склонение существительных |
| `ЕстьНедопустимыеСимволы()` | Проверка строки на спецсимволы | Валидация ввода |
| `ТолькоЦифрыВСтроке()` | Проверка — строка состоит только из цифр? | Валидация ИНН, КПП, номеров |
| `РазложитьСтрокуВМассивПодстрок()` | Разбиение строки по разделителю | Парсинг данных |

### Пример: ПодставитьПараметрыВСтроку

```bsl
// Формирование сообщения с подстановкой параметров
ТекстСообщения = СтроковыеФункцииКлиентСервер.ПодставитьПараметрыВСтроку(
    НСтр("ru = 'Обработано %1 из %2 документов. Ошибок: %3.'"),
    Формат(Обработано, "ЧГ="),
    Формат(Всего, "ЧГ="),
    Формат(КоличествоОшибок, "ЧГ="));
```

### Пример: СтрокаСЧисломПредметов

```bsl
// Корректное склонение: «1 документ», «2 документа», «5 документов»
ТекстОповещения = СтроковыеФункцииКлиентСервер.СтрокаСЧисломПредметов(
    КоличествоДокументов,
    НСтр("ru = 'документ, документа, документов'"));
// Результат: "5 документов" или "1 документ" или "23 документа"
```

## Правило 3: ОбщегоНазначенияКлиентСервер — утилиты для обеих сред

**Почему:** `ОбщегоНазначенияКлиентСервер` — модуль с директивой `&НаКлиентеНаСервереБезКонтекста`, его функции доступны и на клиенте, и на сервере. Это позволяет переиспользовать код без дублирования.

### Ключевые функции

| Функция | Описание |
|---------|----------|
| `ДополнитьТаблицуИзМассива()` | Заполнение таблицы значений из массива |
| `ДополнитьМассив()` | Объединение двух массивов |
| `ДополнитьСтруктуру()` | Объединение двух структур |
| `СвойствоСтруктуры()` | Безопасное чтение свойства (если нет — значение по умолчанию) |
| `ПроверитьПараметр()` | Валидация типа параметра с информативной ошибкой |
| `Сообщить()` | Обёртка для `ОбщегоНазначения.СообщитьПользователю()` (клиентский вариант) |

### Пример: СвойствоСтруктуры — безопасный доступ

```bsl
// ❌ ПЛОХО: ошибка если свойства нет в структуре
Значение = ПараметрыОтчёта.ДатаНачала; // Ошибка если нет свойства «ДатаНачала»

// ✅ ПРАВИЛЬНО: безопасный доступ с значением по умолчанию
ДатаНачала = ОбщегоНазначенияКлиентСервер.СвойствоСтруктуры(
    ПараметрыОтчёта, "ДатаНачала", НачалоГода(ТекущаяДатаСеанса()));
```

## Правило 4: Стратегия поиска функций БСП

**Почему:** БСП содержит тысячи функций, распределённых по сотням модулей. Без стратегии поиска легко пропустить готовую функцию и написать свою, дублирующую.

### Алгоритм поиска

1. **LSP (навигация по символам)** — если доступен LSP-сервер (capability: `navigate_symbol`), ищем по имени:
   ```
   navigate_symbol("СообщитьПользователю")
   navigate_symbol("ЗначенияРеквизитовОбъекта")
   ```

2. **Поиск по тексту модулей** — если LSP недоступен, ищем в исходниках:
   ```
   grep -r "Функция.*КурсВалюты" src/CommonModules/
   ```

3. **AI-ассистент (ai-proxy)** — если прямой поиск не дал результата:
   ```
   ask_ai_assistant("Есть ли в БСП функция для получения курса валюты на дату?")
   ```

4. **Документация ИТС** — для понимания семантики и ограничений найденной функции

### Когда писать своё vs использовать БСП

| Ситуация | Решение |
|----------|---------|
| В БСП есть точно подходящая функция | **Используй БСП** |
| В БСП есть похожая, но с лишним функционалом | **Используй БСП** — лишний функционал не мешает, а код стандартизирован |
| Нужна функция, которой нет в БСП | **Пиши своё**, но в стиле БСП (комментарии, именование) |
| Функция БСП работает, но неоптимально для твоего случая | **Используй БСП** — оптимизируй, только если это bottleneck |
| Конфигурация без БСП | **Пиши своё** или подключай отдельные модули БСП |

## Правило 5: Работа с журналом регистрации через БСП

**Почему:** БСП предоставляет обёртки для `ЗаписьЖурналаРегистрации()`, которые:
- Автоматически определяют метаданные объекта
- Корректно обрабатывают транзакции (запись в ЖР внутри транзакции может быть потеряна при откате)
- Форматируют сообщения по стандарту

### Правильно — через БСП

```bsl
// Логирование ошибки через обёртку БСП
Попытка
    ДокументОбъект.Записать(РежимЗаписиДокумента.Проведение);
Исключение
    ЗаписьЖурналаРегистрации(
        НСтр("ru = 'Проведение документа'",
            ОбщегоНазначения.КодОсновногоЯзыка()),
        УровеньЖурналаРегистрации.Ошибка,
        ДокументОбъект.Метаданные(),
        ДокументОбъект.Ссылка,
        ПодробноеПредставлениеОшибки(ИнформацияОбОшибке()));
    
    ВызватьИсключение;
КонецПопытки;
```

### Стандартные имена событий

```bsl
// Имена событий — иерархические, через точку
// Подсистема.Событие или Объект.Действие
ИмяСобытия = НСтр("ru = 'ОбменДанными.ОтправкаДанных'");
ИмяСобытия = НСтр("ru = 'Проведение документа.Ошибка'");
ИмяСобытия = НСтр("ru = 'ЗагрузкаДанных.ИзВнешнегоИсточника'");
```

## Правило 6: РаботаСФайлами — вместо прямого ФайловаяСистема

**Почему:** Прямая работа с файловой системой через `Новый Файл()`, `КопироватьФайл()`, `ЗначениеВФайл()` не учитывает:
- Права доступа (файловая система vs права 1С)
- Антивирусные проверки
- Временные файлы (могут не удалиться)
- Кросс-платформенность (Windows/Linux)

БСП предоставляет модуль `РаботаСФайлами`, который обрабатывает все эти случаи.

### Пример: получение временного файла

```bsl
// ❌ ПЛОХО: прямое создание файла — можно забыть удалить
ИмяВременногоФайла = ПолучитьИмяВременногоФайла("xlsx");
ТабличныйДокумент.Записать(ИмяВременногоФайла, ТипФайлаТабличногоДокумента.XLSX);
// ... работа с файлом ...
// Если ошибка — файл останется навсегда

// ✅ ПРАВИЛЬНО: через БСП — временные файлы управляются автоматически
ИмяВременногоФайла = ПолучитьИмяВременногоФайла("xlsx");
Попытка
    ТабличныйДокумент.Записать(ИмяВременногоФайла, ТипФайлаТабличногоДокумента.XLSX);
    // ... работа с файлом ...
Исключение
    // Обработка ошибки
КонецПопытки;

// Явное удаление
УдалитьФайлы(ИмяВременногоФайла);
```

## Правило 7: Типовые паттерны БСП для конкретных задач

### Паттерн: Проверка заполнения (ОбработкаПроверкиЗаполнения)

```bsl
// Модуль объекта — стандартный обработчик проверки заполнения
Процедура ОбработкаПроверкиЗаполнения(Отказ, ПроверяемыеРеквизиты)
    
    // Добавляем свои проверки к стандартным
    Если НЕ ЗначениеЗаполнено(Контрагент) Тогда
        ОбщегоНазначения.СообщитьПользователю(
            НСтр("ru = 'Не заполнен контрагент.'"),
            ЭтотОбъект, "Контрагент",, Отказ);
    КонецЕсли;
    
    // Условное исключение реквизитов из проверки
    // (например, если вид операции не требует склада)
    Если ВидОперации = Перечисления.ВидыОпераций.Услуга Тогда
        ОбщегоНазначенияКлиентСервер.УдалитьЗначениеИзМассива(
            ПроверяемыеРеквизиты, "Склад");
    КонецЕсли;
    
КонецПроцедуры
```

### Паттерн: Получение данных для печати

```bsl
// Использование модуля УправлениеПечатью для формирования печатных форм
Процедура Печать(МассивОбъектов, ПараметрыПечати, КоллекцияПечатныхФорм,
        ОбъектыПечати, ПараметрыВывода) Экспорт
    
    Если УправлениеПечатью.НужноПечататьМакет(КоллекцияПечатныхФорм, "Счёт") Тогда
        
        ТабличныйДокумент = Новый ТабличныйДокумент;
        ТабличныйДокумент.КлючПараметровПечати = "Документ.РеализацияТоваровУслуг.Счёт";
        
        // ... формирование печатной формы ...
        
        УправлениеПечатью.ВывестиТабличныйДокументВКоллекцию(
            КоллекцияПечатныхФорм, "Счёт", НСтр("ru = 'Счёт на оплату'"),
            ТабличныйДокумент);
    КонецЕсли;
    
КонецПроцедуры
```

## Правило 8: Не дублируйте функционал БСП

**Почему:** Каждая самописная функция — это код, который нужно поддерживать, тестировать и обновлять. БСП уже делает это за вас. Дублирование приводит к:
- Двум источникам истины — ваш код vs БСП
- Конфликтам при обновлении конфигурации
- Упущенным edge cases (БСП учитывает их, ваш код — вероятно, нет)

### Распространённые случаи дублирования

| Что часто пишут сами | Что есть в БСП |
|----------------------|----------------|
| Функция для получения реквизита по ссылке | `ОбщегоНазначения.ЗначениеРеквизитаОбъекта()` |
| Проверка заполненности значения | `ЗначениеЗаполнено()` (платформа) + `ОбщегоНазначенияКлиентСервер.ПроверитьПараметр()` |
| Формирование строки с подстановкой | `СтроковыеФункцииКлиентСервер.ПодставитьПараметрыВСтроку()` |
| Склонение слов | `СтроковыеФункцииКлиентСервер.СтрокаСЧисломПредметов()` |
| Отправка почты | `РаботаСПочтовымиСообщениями` |
| Получение курса валюты | `РаботаСКурсамиВалют.ПолучитьКурсВалюты()` |
| Логирование в журнал | Стандартный `ЗаписьЖурналаРегистрации()` по правилам БСП |

## Правило 9: Модули «КлиентСервер» — разделение ответственности

**Почему:** В БСП модули организованы по принципу окружения выполнения:

| Суффикс модуля | Среда | Пример |
|----------------|-------|--------|
| (без суффикса) | Сервер | `ОбщегоНазначения` |
| `Клиент` | Клиент | `ОбщегоНазначенияКлиент` |
| `КлиентСервер` | Обе среды | `ОбщегоНазначенияКлиентСервер` |
| `Сервер` (явно) | Только сервер | `СтандартныеПодсистемыСервер` |
| `ПовтИсп` | Сервер, с кэшированием | `ОбщегоНазначенияПовтИсп` |

**Правила выбора:**
- Для клиентского кода формы — ищите сначала в `*КлиентСервер`, потом в `*Клиент`
- Для серверного кода — ищите в основном модуле (без суффикса) или `*Сервер`
- `*ПовтИсп` — функции с кэшированием результата на время сеанса, используйте для часто запрашиваемых справочных данных

### Пример: правильный выбор модуля

```bsl
// На клиенте — используем КлиентСервер модуль
&НаКлиенте
Процедура КонтрагентПриИзменении(Элемент)
    // ОбщегоНазначенияКлиентСервер — доступен на клиенте
    Если НЕ ЗначениеЗаполнено(Объект.Контрагент) Тогда
        Возврат;
    КонецЕсли;
    
    // Для серверного вызова используем БезКонтекста
    ДанныеКонтрагента = ПолучитьДанныеКонтрагентаНаСервере(Объект.Контрагент);
КонецПроцедуры

// На сервере — используем серверный модуль
&НаСервереБезКонтекста
Функция ПолучитьДанныеКонтрагентаНаСервере(КонтрагентСсылка)
    // ОбщегоНазначения — серверный модуль
    Возврат ОбщегоНазначения.ЗначенияРеквизитовОбъекта(
        КонтрагентСсылка, "Наименование, ИНН, КПП");
КонецФункции
```
