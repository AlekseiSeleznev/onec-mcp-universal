# /openspec-archive — Архивация feature-изменения

Используется после завершения работ по `tasks.md` для очистки рабочего потока и закрепления истории.

## Когда применять

- Все задачи в `tasks.md` отмечены как `[x]`,
- Пройдены проверки (в том числе профильные `*-validate`/`validate_query`),
- Пользователь подтверждает закрытие.

## Процесс

1. Определи `feature-id`:
   - из аргумента или из структуры `openspec/changes/`.
2. Проверить статус:
   - все элементы в `openspec/changes/<feature-id>/tasks.md` закрыты;
   - если есть незакрытые задачи — остановись и верни статус.
3. Перенести каталог в архив:
   - `openspec/changes/<feature-id>/` → `openspec/changes/archive/<feature-id>/`.
4. Если есть `openspec/specs/` и `spec`-дельты, аккуратно обнови целевые спецификации.
   - Иначе зафиксируй причину отсутствия.
5. Обнови отчёт:
   - добавь финальную строку о статусе закрытия в `session-notes.md` (если файл есть).

## Результат

- `feature` перемещён в `openspec/changes/archive/`,
- выполненные спецификации/заметки закреплены,
- пользователю возвращен сводный статус.

## Запреты

- Не архивировать незавершённые изменения.
- Не удалять исходники изменения при архивировании.
