# /1c-help-mcp — Поиск по документации 1С

Используй для быстрого уточнения документации и API перед реализацией.

## Когда использовать

- Нужна сигнатура метода/функции платформы.
- Нужно проверить правильность синтаксиса выражения.
- Требуются краткие ссылки в документации по справочнику API.

## Когда не использовать

- Для поиска по коду конфигурации — используйте `bsl_search_tool`/`bsl_search`.
- Для проверки бизнес-логики — используйте `get_bsl_syntax_help`/`info` по типам/методам.

## Рекомендуемый порядок

1. Сначала локализуй предмет поиска:
   - точное имя метода/функции;
   - или краткая задача (`"работа с файлами"` и т.д.).
2. Выполни поиск по API/помощи:
   - `get_bsl_syntax_help` для метода/метода + типа;
   - `search` в случае свободного запроса по платформенной терминологии.
3. Если требуется кодовый пример, задай контекст запроса в отдельную задачу и проверь
   вывод через валидацию/тесты перед применением.
4. Перед генерацией кода явно закрой список критериев: что считаешь корректным ответом.

## Практическое правило

- Не писать решение «вслепую»: сначала подтверждение по документации, затем
  план/реализация через профильные скилы.
- Любые решения, связанные с API, сопровождай ссылкой на найденый API-элемент.

## Формат ответа

- что искал (query/tool)
- какие разделы API нашли
- чем это закрывает задачу
- какой следующий шаг (переход к `use-bsp`, `query-writing`, `1c-query-opt` и т.п.)
