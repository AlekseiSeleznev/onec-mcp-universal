# /validate

Оркестратор проверок. Выбирает профильный validate-скилл и запускает его.

## Маршрутизация

- Конфигурация -> `/cf-validate`
- Расширение -> `/cfe-validate`
- Объект метаданных -> `/meta-validate`
- Форма -> `/form-validate`
- СКД -> `/skd-validate`
- MXL -> `/mxl-validate`
- Роль -> `/role-validate`
- Подсистема -> `/subsystem-validate`
- Командный интерфейс подсистемы -> `/interface-validate`
- EPF/ERF -> `/epf-validate` / `/erf-validate`

## Правила

1. Определи тип целевого артефакта перед запуском валидации.
2. Не подменяй профильную валидацию общими проверками.
3. Для связанных изменений (например, объект + форма) запускай validate для каждого затронутого типа.
4. В ответе всегда фиксируй результат: passed/failed + ключевая причина.
