# /inspect

Оркестратор для inspection-задач. Маршрутизирует к существующим `*-info` скилам.

## Маршрутизация

- Конфигурация -> `/cf-info`
- Объект метаданных -> `/meta-info`
- Управляемая форма -> `/form-info`
- СКД -> `/skd-info`
- Макет MXL -> `/mxl-info`
- Роль -> `/role-info`
- Подсистема -> `/subsystem-info`

## Правила

1. Сначала классифицируй тип объекта, который нужно проанализировать.
2. Используй ровно один профильный info-скилл, если запрос не комплексный.
3. Для комплексного запроса выполняй анализ по очереди и в конце давай объединённую сводку.
4. Если путь/имя неоднозначны — запроси уточнение перед запуском.
