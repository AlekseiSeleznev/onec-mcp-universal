# /1c-web-session — Веб-сессия 1С (совместимый вход)

Алиас-оркестратор для внешних сценариев, где ранее применялся `1c-web-session`.

## Базовый режим

1. Читай URL и параметры публикации из проекта (`AGENTS.md`, `README` / локальный `web` URL).
2. Запускай браузерную автоматизацию через `/web-test`.
3. Встроенными сценариями выполняй:
   - вход в публикацию,
   - навигацию по разделам/командам,
   - создание/правку объектов,
   - чтение форм/таблиц для smoke-проверок.

## Что считать критичным

- критичные диалоги (включить/подтвердить/отменить/сохранить),
- ошибки валидации,
- ошибки сериализации форм/выполнения команд,
- несоответствие контрольных значений после записи.

## Правила

1. По возможности держи сценарии в формате "action → verify" (чтобы воспроизводить).
2. Для сложных сценариев указывай fallback:
   - подготовка данных через backend-скиллы,
   - проверка UI через `/web-test`.
3. После выполнения — короткий отчёт по шагам и статусам.
