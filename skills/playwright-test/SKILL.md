# /playwright-test — Playwright UI сценарии (совместимость)

В этом стекe это совместимая обёртка над `/web-test`:

- запуск сценариев;
- сценарное чтение форм и таблиц;
- навигация по разделам и командам;
- формирование отчёта шага выполнения.

## Как использовать

1. Получи сценарий пользователя/теста:
   - "что проверяем" и "в каком участке конфигурации".
2. Запускай через `/web-test` существующими примерами или inline-сценарий.
3. После исполнения приводи отчёт:
   - success/fail по шагам,
   - какие ошибки были в форме/модалках,
   - что нужно зафиксировать как регресс-тест.

## Правила

1. Не дублируй сценарий, если уже покрыт существующим `/web-test` flow.
2. Для не-UI проверок не используйте `playwright-test`; лучше `/query-writing`, `validate_query`, `mxl-validate`, `cf-validate`.
3. Любой новый сценарий должен иметь наблюдаемый критерий pass/fail.
