# /session-retro

Краткая ретроспектива по итогам сессии с сохранением в `session-notes.md`.

## Поведение

1. Сформируй 5-10 пунктов по итогам:
   - что сработало,
   - что замедлило,
   - какие решения стоит повторять,
   - какие ошибки не повторять.
2. Если `session-notes.md` существует — добавь секцию `## Retrospective`.
3. Если не существует — создай файл с базовой структурой и секцией ретроспективы.

## Правила

- Только actionable выводы.
- Не раздувай отчёт: кратко и по делу.
