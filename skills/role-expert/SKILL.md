# /role-expert — Роли 1С (совместимый вход)

Ключевой оркестратор для работы с ролями в стиле внешнего набора скиллов:

- создать роль по JSON-подобным или явным параметрам через `/role-compile`;
- изменить существующие права через `/meta-edit` по `Roles/...` и/или `rights`;
- проверить итог через `/role-validate`.

## Режимы

### 1) Создать новую роль
Собери параметры:
- имя роли,
- объектный набор,
- тип прав,
- ограничения.

Затем применяй:
- `/role-compile` для генерации/обновления XML и регистрации роли в `Configuration.xml`.

### 2) Изменить существующую роль
- `/inspect` для анализа текущих прав;
- `/meta-edit` для точечных изменений;
- `/role-validate` для проверки целостности и прав.

### 3) Проверить роль после изменений
- `/role-validate` обязателен.
- при затрагивании объектов конфигурации дополнительно `/meta-validate` или `/validate` для связанных артефактов.

## Правила

1. Роль создаётся в одном проходе: анализ → изменение/сборка → валидация.
2. Не смешивать изменение ролей с архитектурными изменениями, которые ещё не оформлены через `/openspec-proposal` или аналог.
3. Для сложных RLS/ограничений сначала зафиксируй матрицу прав и критерии в `proposal.md` или `design.md`.
