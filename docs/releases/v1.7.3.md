# onec-mcp-universal v1.7.3

## Что вошло

- `bsl-graph` теперь поставляется как встроенный профиль репозитория:
  - локально собирается из `bsl-graph-lite/`;
  - поднимается одной командой через `./setup.sh --with-bsl-graph` или `docker compose --profile bsl-graph up -d --build`;
  - больше не зависит от внешнего образа `ghcr.io/alkoleft/bsl-graph:latest`.
- Удаление базы из дашборда стало жёстко очищать runtime-состояние:
  - останавливаются и удаляются toolkit/LSP контейнеры базы;
  - очищаются export/index task state и in-memory BSL search state;
  - база исчезает из graph dataset после rebuild;
  - физические BSL-файлы на диске сохраняются.
- `disconnect_database` закреплён как soft-disconnect:
  - runtime останавливается;
  - запись базы остаётся в registry;
  - graph/state не уничтожаются.
- Закрыты runtime-дефекты, всплывшие на полном аудите:
  - `platform-context` больше не роняет gateway session на `search`;
  - `did_change_watched_files` больше не роняет gateway session при stale LSP stdio session;
  - `graph_related` корректно работает с URL-encoded `object_id`, включая кириллицу и пути;
  - bundled `bsl-graph` индексирует только базы из registry state.
