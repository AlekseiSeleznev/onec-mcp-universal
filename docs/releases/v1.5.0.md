# onec-mcp-universal v1.5.0

Дата: 2026-04-13

## Highlights

- `export_bsl_sources` переведён в безопасный режим: при недоступном host export service теперь `hard fail`.
- Контейнерный `1cv8 DESIGNER` включается только явным флагом `ALLOW_CONTAINER_DESIGNER_EXPORT=true`.
- Убран хрупкий хардкод DB-tool names: routing теперь динамический по реально подключённым backend tools.
- Улучшены эксплуатационные cleanup-пути:
  - очистка orphan per-DB контейнеров на старте;
  - очистка terminated MCP sessions (и связанных session routing записей).
- Усилена потокобезопасность `DatabaseRegistry` (`mark_epf_connected` под lock).
- Запись `db_state.json` переведена на атомарный `tmp + replace`.
- Добавлен optional structured logging (`LOG_JSON=true`).
- Начата декомпозиция монолитного Web UI: docs-блок вынесен в отдельный модуль `web_docs.py`.

## Testing

- Добавлены/обновлены unit + smoke + load-smoke тесты.
- Локальный контур стабилизирован через `gateway/requirements-dev.txt` и `gateway/pytest.ini` (`asyncio_mode = auto`).
- Полный прогон: `418 passed`.

## Compatibility notes

- Для безопасной работы BSL export рекомендуется настроить `EXPORT_HOST_URL`.
- Новый флаг:
  - `ALLOW_CONTAINER_DESIGNER_EXPORT=false` (по умолчанию)
- Новый опциональный флаг логирования:
  - `LOG_JSON=false` (по умолчанию)
