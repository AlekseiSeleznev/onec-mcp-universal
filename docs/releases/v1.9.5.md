# onec-mcp-universal v1.9.5

`v1.9.5` — точечный security hotfix для `docker-control` без смены общей runtime-архитектуры.

## Что исправлено

- `docker-control` больше не принимает lifecycle/API-запросы анонимно.
- Добавлен общий bearer token `DOCKER_CONTROL_TOKEN` для связи `gateway -> docker-control`.
- `setup.sh` автоматически генерирует токен, если его нет в `.env`.
- `verify-install.sh` и `verify-install.ps1` проверяют не только `GET /health`, но и то, что защищённый endpoint sidecar без токена возвращает `401`.
- `lsp_start_api` теперь валидирует `mount_path`: абсолютный путь, без traversal и без запрещённых символов; путь должен оставаться внутри настроенного BSL workspace.
- Внутри `docker-control` убрана shell-интерполяция для проверки содержимого каталога в LSP-контейнере.
- `write_env_api` принимает только валидный `.env`-формат: пустые строки, комментарии и `KEY=VALUE`.

## Дашборд и документация

- Редактор `.env` в дашборде маскирует `DOCKER_CONTROL_TOKEN`, но сохраняет его при обычном редактировании файла.
- README, `CODEX.md` и dashboard docs обновлены под новый auth-контракт sidecar.

## Проверка

- `GET http://127.0.0.1:8091/health` остаётся открытым.
- `POST/GET http://127.0.0.1:8091/api/...` без `Authorization: Bearer ...` возвращает `401`.
- Внешний MCP endpoint не меняется: `http://localhost:8080/mcp`.
