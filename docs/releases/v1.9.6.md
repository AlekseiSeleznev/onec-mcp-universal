# onec-mcp-universal v1.9.6

`v1.9.6` — patch hotfix, который закрывает оставшееся exposure `docker-control` и убирает фиксированный anonymizer salt.

## Что исправлено

- `docker-control` на Linux публикуется только на `127.0.0.1:8091`, а не на всех интерфейсах.
- В `docker-compose.windows.yml` host port `8091` для `docker-control` убран полностью; gateway использует sidecar по внутреннему адресу `http://docker-control:8091`.
- `setup.sh` генерирует новый secret `ANONYMIZER_SALT` и хранит его в `.env`.
- Маскирование ПД больше не использует фиксированный `"onec-mcp-anon"`; salt стал persistent и независимым от `DOCKER_CONTROL_TOKEN`.
- Редактор `.env` в дашборде теперь маскирует и восстанавливает оба секрета: `DOCKER_CONTROL_TOKEN` и `ANONYMIZER_SALT`.
- `verify-install.sh` и `verify-install.ps1` стали OS-aware для проверки `docker-control`: Linux проверяет loopback host port, Windows/macOS — sidecar изнутри `onec-mcp-gw`.

## Документация

- README, `CODEX.md` и dashboard docs обновлены под loopback/internal-only модель `docker-control`.
- Описан новый env-параметр `ANONYMIZER_SALT`.

## Проверка

- MCP endpoint не меняется: `http://localhost:8080/mcp`.
- `GET /health` у `docker-control` остаётся без auth.
- `/api/*` у `docker-control` по-прежнему требуют `Authorization: Bearer <DOCKER_CONTROL_TOKEN>`.
