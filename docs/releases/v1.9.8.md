# onec-mcp-universal v1.9.8

`v1.9.8` — follow-up patch, который публикует финальное состояние hotfix-цикла после `v1.9.7` без изменения runtime-контракта.

## Что вошло

- Runtime-исправление `docker-control` fallback policy из `v1.9.7` остаётся без изменений:
  sidecar auth/config/runtime errors больше не маскируются direct-Docker fallback'ом.
- `disconnect` по-прежнему возвращает реальную ошибку runtime stop вместо ложного success.
- Reconnect/register path по stale managed `project_path` остаётся исправленным.

## Что изменилось в этом patch

- Дотянуты branch tests для новых helper-веток в `docker_manager.py` и `server.py`.
- Локальный canonical test run теперь совпадает с GitHub CI coverage gate.
- Финальный `gateway/tests` в coverage-режиме: `1031 passed`, `100.00% coverage`.

## Проверка

- MCP endpoint не меняется: `http://localhost:8080/mcp`.
- `gateway` продолжает работать через `docker-control`, а не через прямой `docker.sock`.
- Release нужен для того, чтобы GitHub `main`, semver tag и published Docker package указывали на один и тот же финальный commit hotfix-цикла.
