# v1.10.0 — Universal 1C Report Execution

## Added

- New multi-database report catalog stored in `/data/report-catalog.sqlite`.
- New MCP tools: `analyze_reports`, `enrich_report_docs`, `find_reports`, `list_reports`, `describe_report`, `run_report`, `get_report_result`, `explain_report_strategy`.
- User-facing report resolution: users and LLMs can request reports by UI title, for example `Расчетный листок`, without knowing technical names like `Отчет.АнализНачисленийИУдержаний`.
- Optional report documentation enrichment from ITS/1C:Naparnik improves search for accountant-facing phrases, for example `расчетка сотрудника`.
- Dashboard reports tab for catalog analysis, report search, execution and JSON result inspection.
- First built-in complex report adapter for `Расчетный листок` / `АнализНачисленийИУдержаний`.

## Notes

- Report tools require explicit `database` to keep concurrently connected bases isolated.
- `bsl-graph` is used for analysis and strategy ranking only; report execution goes through per-database toolkit/EPF.
- Large report results are stored as `.json.gz` files under `/data/report-results/<database>/`.
