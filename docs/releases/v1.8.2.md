# onec-mcp-universal v1.8.2

## Что вошло

- deterministic настройка `BSL_WORKSPACE` и `BSL_HOST_WORKSPACE` через `setup.sh` на чистой машине
- `bsl-graph-lite` получил встроенный `HEALTHCHECK`
- из `docker-compose.yml` удалён legacy NebulaGraph stack `bsl-graph-legacy`
- GitHub publish теперь публикует не только gateway image, но и отдельный bundled graph image
- документация синхронизирована с кодом: HTTPS clone, безопасное удаление skills, динамическое описание числа тестов

## Технические изменения

- `.env.example`, `setup.sh`, `tools/export-host-service.py` приведены к единым правилам workspace path
- `.github/workflows/ci.yml` теперь использует pip cache
- `.github/workflows/docker-publish.yml` публикует:
  - `ghcr.io/alekseiseleznev/onec-mcp-universal`
  - `ghcr.io/alekseiseleznev/onec-mcp-universal-bsl-graph-lite`

## Проверка

- полный `pytest gateway/tests` с coverage gate `100%`
- destructive reinstall из GitHub
- `./setup.sh --with-bsl-graph`
- `./verify-install.sh`

## Совместимость

- MCP endpoint не меняется: `http://localhost:8080/mcp`
- `./setup.sh` остаётся основным entrypoint
- ломающие security-изменения вынесены в отдельный следующий трек
