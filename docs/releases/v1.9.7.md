# onec-mcp-universal v1.9.7

`v1.9.7` — patch hotfix, который завершает жёсткую политику работы `gateway -> docker-control` и закрывает ложные direct-Docker fallback'и после `v1.9.6`.

## Что исправлено

- `gateway` больше не считает sidecar auth/config/runtime errors поводом для direct-Docker fallback.
- `disconnect` теперь корректно возвращает ошибку, если runtime нельзя остановить через `docker-control`, вместо ложного success.
- Для sidecar-unreachable сценариев direct fallback сохранён, но при его провале ошибка теперь содержит обе причины: sidecar и direct fallback.
- Политика fallback унифицирована для lifecycle и diagnostic wrappers: `stop_db_containers`, orphan cleanup, Docker system info, container info/logs, restart и image presence.
- Исправлен reconnect/register regression для stale managed `project_path`: старые значения вроде `/home/as` нормализуются обратно под текущий BSL workspace root.

## Тесты

- Добавлены regression tests на sidecar runtime/config errors без direct fallback.
- Добавлены tests на combined error при sidecar-unreachable + failed direct fallback.
- Добавлен regression test на `disconnect`, который больше не возвращает success при сломанном control plane.
- Полный `gateway/tests` после изменений: `1024 passed`.

## Проверка

- MCP endpoint не меняется: `http://localhost:8080/mcp`.
- `gateway` по-прежнему управляет Docker runtime только через `docker-control`.
- Sidecar runtime/config errors теперь должны быть видны пользователю и в диагностике, а не маскироваться сообщением `docker SDK is not available`.
