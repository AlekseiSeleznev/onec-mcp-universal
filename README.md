# onec-mcp-universal

Единый MCP-сервер для работы с 1С:Предприятие из AI-ассистентов (Claude Code, Cursor, Windsurf).

Один сервер вместо нескольких отдельных MCP-подключений. Шлюз принимает все запросы и маршрутизирует их к нужному бэкенду автоматически.

## Что умеет

**37 инструментов и 1 ресурс:**

| Категория | Кол-во | Описание |
|---|---|---|
| Данные 1С | 8 | Запросы к БД, выполнение кода, метаданные, журнал регистрации, права доступа |
| Документация платформы | 5 | Поиск по API 1С, методы типов, конструкторы |
| Навигация по BSL | 14 | Поиск символов, hover-подсказки, диагностика, структура файлов |
| Шлюз | 10 | Подключение/переключение баз, валидация запросов, граф связей, статус |

**MCP Resource:** справочник синтаксиса встроенного языка 1С (`syntax_1c.txt`) — AI использует его как контекст при написании BSL-кода.

## Требования

- Docker 24+ и Docker Compose v2
- Linux (Ubuntu 22.04+) или Windows 10/11 с Docker Desktop (WSL2)
- Платформа 1С:Предприятие 8.3, установленная на хосте
- Доступ к информационной базе 1С (серверная или файловая)

## Установка

### Шаг 1. Скачать проект

```bash
git clone https://github.com/AlekseiSeleznev/onec-mcp-universal.git
cd onec-mcp-universal
```

### Шаг 2. Создать файл настроек

```bash
cp .env.example .env
```

Открыть файл `.env` в любом текстовом редакторе и указать путь к установленной платформе 1С:

```env
PLATFORM_PATH=/opt/1cv8/x86_64/8.3.27.2074
```

Чтобы узнать путь к платформе, выполнить: `ls /opt/1cv8/x86_64/`.

### Шаг 3. Запустить сервис выгрузки исходников

Этот сервис работает на хосте (не в Docker) и нужен для выгрузки конфигурации 1С в файлы BSL.

Открыть отдельный терминал и выполнить:

```bash
python3 tools/export-host-service.py --port 8082 --workspace ~/1c-projects
```

Параметр `--workspace` — папка, куда будут сохраняться исходники. Можно указать любой удобный путь.

Терминал оставить открытым. Для постоянной работы рекомендуется добавить сервис в systemd.

### Шаг 4. Запустить контейнеры

```bash
docker compose up -d
```

Первый запуск занимает 2-3 минуты: скачиваются образы контейнеров и собирается шлюз.

### Шаг 5. Проверить

```bash
curl http://localhost:8080/health
```

Ожидаемый ответ:

```json
{
  "status": "ok",
  "backends": {
    "onec-toolkit": {"ok": true, "tools": 8},
    "platform-context": {"ok": true, "tools": 5},
    "bsl-lsp-bridge": {"ok": true, "tools": 14}
  }
}
```

Если все три бэкенда показывают `"ok": true` — установка завершена.

## Подключение к AI-ассистенту

### Cursor

1. Открыть **Settings** → **MCP**.
2. Нажать **Add new MCP server**.
3. Заполнить:
   - Name: `onec`
   - Type: `HTTP`
   - URL: `http://localhost:8080/mcp`
4. Нажать **Save**.

В списке инструментов появятся все инструменты с префиксом `onec-universal__`.

### Claude Code

Выполнить в терминале:

```bash
claude mcp add onec --transport http http://localhost:8080/mcp
```

### Другие AI-ассистенты

Любой MCP-клиент с поддержкой Streamable HTTP может подключиться по адресу:

```
http://localhost:8080/mcp
```

## Подключение базы 1С

Прежде чем работать с данными конкретной базы, её необходимо подключить к шлюзу. Подключение выполняется один раз после каждого перезапуска шлюза.

### Шаг 1. Подключить базу через AI

Написать в чате с AI:

```
Подключи базу ERP, строка подключения Srvr=myserver;Ref=erp_demo;, папка проекта /home/user/1c-projects/ERP
```

AI вызовет инструмент `connect_database`. Шлюз автоматически создаст два Docker-контейнера для этой базы.

**Форматы строки подключения:**

| Тип базы | Строка подключения |
|---|---|
| Серверная | `Srvr=имя_сервера;Ref=имя_базы;` |
| С логином и паролем | `Srvr=имя_сервера;Ref=имя_базы;Usr=admin;Pwd=пароль;` |
| Файловая | `File=/путь/к/базе` |

### Шаг 2. Открыть обработку MCPToolkit в 1С

1. В клиенте 1С:Предприятие открыть внешнюю обработку `1c/MCPToolkit.epf`.
2. При запросе безопасности — разрешить открытие.
3. Обработка автоматически подключится к шлюзу. В строке состояния появится текст **«Подключено»**.

Обработку необходимо держать открытой во время работы с AI. Она является посредником между шлюзом и информационной базой 1С.

### Шаг 3. Выгрузить исходники BSL

Написать AI:

```
Выгрузи исходники конфигурации
```

Или нажать кнопку **«Выгрузить BSL»** в обработке.

После выгрузки BSL Language Server начнёт индексацию файлов. На крупных конфигурациях (ERP 2.5, ЗУП 3) индексация занимает 3-5 минут. Чтобы проверить готовность, написать AI:

```
Покажи статус BSL индексации
```

Когда индексация завершится — все инструменты навигации по коду станут доступны.

## Работа с несколькими базами

К одному шлюзу можно подключить несколько информационных баз и переключаться между ними.

**Подключить вторую базу:**

```
Подключи базу ZUP, строка подключения Srvr=myserver;Ref=zup;, папка проекта /home/user/1c-projects/ZUP
```

**Переключиться на другую базу:**

```
Переключись на базу ZUP
```

**Посмотреть список подключённых баз:**

```
Покажи список подключённых баз 1С
```

**Отключить базу:**

```
Отключи базу ZUP
```

При отключении шлюз остановит и удалит Docker-контейнеры этой базы. Данные базы 1С и исходники BSL при этом не затрагиваются.

## Опциональные модули

Базовая установка включает три основных бэкенда. Дополнительные модули запускаются через профили Docker Compose.

### Тестирование YaXUnit (mcp-onec-test-runner)

Запуск тестов, сборка проекта и проверка синтаксиса из AI-ассистента. Использует [mcp-onec-test-runner](https://github.com/alkoleft/mcp-onec-test-runner).

**Настройка:**

1. Скопировать и отредактировать конфигурацию:

```bash
cp test-runner/application.yml.example test-runner/application.yml
# Открыть test-runner/application.yml и указать строку подключения и путь к проекту
```

2. Запустить контейнер:

```bash
docker compose --profile test-runner up -d
```

3. Добавить в `.env` и перезапустить шлюз:

```env
ENABLED_BACKENDS=onec-toolkit,platform-context,bsl-lsp-bridge,test-runner
```

```bash
docker compose restart gateway
```

**Примеры использования:**

```
Запусти все тесты YaXUnit
```

```
Запусти тесты модуля ОбработкаЗаказов
```

```
Собери проект конфигурации
```

## Установка на Windows

На Windows платформа 1С работает только на хосте, а не внутри Linux-контейнеров. Для выгрузки BSL необходимо запустить отдельный сервис.

### Шаг 1. Запустить сервис выгрузки

Открыть командную строку или PowerShell и выполнить:

```cmd
python tools\export-host-service.py --port 8082 --workspace C:\1c-projects
```

### Шаг 2. Настроить переменные окружения

В файле `.env` добавить:

```env
EXPORT_HOST_URL=http://host.docker.internal:8082
```

### Шаг 3. Запустить контейнеры

```bash
docker compose up -d
```

Если в логах `platform-context` появляется ошибка монтирования `/opt/1cv8` — закомментировать или удалить строку `${HOST_PLATFORM_PATH:-/opt/1cv8}:/opt/1cv8:ro` в секции `platform-context` → `volumes` файла `docker-compose.yml`.

Остальные шаги (подключение к AI, подключение базы) аналогичны описанным выше. Вместо Linux-путей использовать Windows-пути (`C:\1c-projects\ERP`).

## Архитектура

```
Cursor / Claude Code / Windsurf
       | HTTP :8080/mcp
       v
+-------------------------------------+
|  onec-mcp-gw  (Python, host network)|
|                                      |
|  Статические бэкенды:                |
|  - onec-mcp-toolkit   :6003         |  <- Streamable HTTP
|  - onec-mcp-platform  :8081         |  <- SSE
|                                      |
|  Динамические бэкенды (per-DB):      |
|  - onec-toolkit-ERP   :6100+        |  <- создаются при connect_database
|  - mcp-lsp-ERP        (stdio)       |  <- docker exec
+-------------------------------------+
         ^
         |  MCPToolkit.epf (клиент 1С)
         |  держит соединение с onec-toolkit
```

**Контейнеры:**

| Контейнер | Образ | Роль |
|---|---|---|
| `onec-mcp-gw` | собирается локально | Шлюз, маршрутизация |
| `onec-mcp-toolkit` | `roctup/1c-mcp-toolkit-proxy` | Статический бэкенд данных |
| `onec-mcp-platform` | `ghcr.io/alkoleft/mcp-bsl-platform-context` | Документация платформы |
| `onec-toolkit-{db}` | `roctup/1c-mcp-toolkit-proxy` | Динамический бэкенд (per-DB) |
| `mcp-lsp-{db}` | `mcp-lsp-bridge-bsl:latest` | BSL Language Server (per-DB) |

## Планы развития

### Анонимизация данных

Маскировка персональных данных в ответах (ФИО, ИНН, СНИЛС) со стабильным маппингом. Снимает барьер 152-ФЗ/GDPR при подключении production-баз к облачным AI. Подход описан в [1c-mcp-toolkit](https://github.com/ROCTUP/1c-mcp-toolkit).

### RAG-поиск по БСП и коду конфигурации

Семантический поиск по исходникам конфигурации и Библиотеке стандартных подсистем. [vibecoding1c.ru](https://vibecoding1c.ru/mcp_server) реализует аналогичный подход через серверы SSLSearchServer и HelpSearchServer.

### Write-операции метаданных

Переименование объектов с каскадным рефакторингом, добавление реквизитов, снимок формы. [DitriXNew/EDT-MCP](https://github.com/DitriXNew/EDT-MCP) реализует 26 инструментов для EDT.

### Поиск по ИТС через 1С:Напарник

Поиск по документации ИТС и типовым конфигурациям через API 1С:Напарника. [Описание подхода](https://infostart.ru/1c/articles/2624226/).

## Удаление

**Остановить и удалить все контейнеры:**

```bash
docker compose down
```

**Удалить собранный образ шлюза:**

```bash
docker rmi onec-mcp-universal-gateway
```

**Удалить проект:**

```bash
cd .. && rm -rf onec-mcp-universal
```

Данные баз 1С и выгруженные исходники BSL при удалении не затрагиваются.

**Удалить подключение MCP из Claude Code:**

```bash
claude mcp remove onec
```
